# Веб-приложение аптеки "Здоровье+"

В данном репозитории содержится исходный код веб-приложения аптеки "Здоровье+", разработанного в рамках курсового проекта. Приложение построено на базе микросервисной архитектуры с разделением серверной части (REST API) и пользовательского интерфейса.

## Архитектура системы

Проект разделен на два независимых компонента:

*   **Бэкенд (API):** Разработан с использованием Java 21 и Spring Boot 3. Отвечает за бизнес-логику, взаимодействие с базой данных и безопасность (аутентификацию и авторизацию).
*   **Фронтенд (UI):** Разработан на Node.js, Express и шаблонизаторе EJS. Отвечает за пользовательский интерфейс и взаимодействует с бэкендом посредством HTTP-запросов.

## Ключевые возможности

*   **Каталог препаратов:** Подробный список медицинских товаров с описанием, ценами и информацией об остатках.
*   **Умный поиск:** Оптимизированный, регистронезависимый алгоритм поиска, который ищет совпадения как по названиям, так и по описаниям препаратов непосредственно на уровне базы данных.
*   **Ожидаемые поступления:** Специальный раздел, отображающий товары, которые скоро поступят в продажу.
*   **Управление доступом на основе ролей (RBAC):**
    *   **Пользователь (User):** Может регистрироваться, просматривать каталог и общаться со службой поддержки.
    *   **Администратор (Admin):** Имеет полный доступ к управлению каталогом (добавление, редактирование, удаление товаров) и управлению будущими поставками.
*   **Чат поддержки (Система тикетов):** Чат в реальном времени, где пользователи могут приватно общаться с администрацией. У администратора есть специальная панель для просмотра и ответа на все активные обращения клиентов.
*   **Безопасная аутентификация:** Использование JSON Web Tokens (JWT) для безопасного управления сессиями без сохранения состояния на сервере (stateless).
*   **Меры безопасности:** Защита от распространенных веб-уязвимостей, включая SQL-инъекции, межсайтовый скриптинг (XSS) и подделку межсайтовых запросов (CSRF). Настроены HTTP-заголовки безопасности с помощью библиотеки Helmet.

## Технологический стек

### Бэкенд
*   **Язык:** Java 21
*   **Фреймворк:** Spring Boot 3
*   **Безопасность:** Spring Security, JWT (jjwt)
*   **База данных:** H2 Database (Файловый режим для сохранения данных)
*   **ORM:** Spring Data JPA (Hibernate)
*   **Сборка:** Maven

### Фронтенд
*   **Среда выполнения:** Node.js
*   **Фреймворк:** Express.js
*   **Шаблонизатор:** EJS (Server-Side Rendering)
*   **HTTP-клиент:** Axios
*   **CSS Фреймворк:** Bootstrap 5.3
*   **Библиотеки безопасности:** Helmet

## Запуск проекта

Для запуска приложения на локальном компьютере вам потребуется два окна терминала (для бэкенда и фронтенда).

### Требования
*   Java Development Kit (JDK) 21
*   Node.js (v18 или выше) и npm

### 1. Запуск Бэкенда

Перейдите в директорию бэкенда и используйте Maven wrapper для запуска приложения Spring Boot.

```bash
cd backend
./mvnw spring-boot:run
```

Сервер API запустится по адресу `http://localhost:8080`.
*Примечание: При первом запуске будет автоматически создан файл базы данных, заполненный тестовыми препаратами и учетной записью администратора по умолчанию.*

### 2. Запуск Фронтенда

В новом окне терминала перейдите в директорию фронтенда, установите зависимости и запустите сервер Node.js.

```bash
cd frontend
npm install
node app.js
```

Веб-интерфейс будет доступен по адресу `http://localhost:3000`.
